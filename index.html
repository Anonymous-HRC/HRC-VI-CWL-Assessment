Overview

This website presents a vision-attentional and behavioral cognitive workload assessment framework for human-robot collaborative assembly.

It highlights the core model framework driven by visual attention and behavioral information, demonstration videos from collaborative assembly scenarios, and the corresponding model evaluation and validation results.

Methodology

The framework estimates cognitive workload in human-robot collaborative assembly by integrating attention orientation, behavioral engagement, context-aware attention switching, and robot-interaction hesitation.

1. Visual-Attentional and Behavioral State Representation

The framework jointly considers where the operator attends and how the operator behaves during collaboration.

\[ \mathbf{o}(t)=\left\{\theta_{h}(t),\mathbf{u}_{b}(t),\mathbf{x}_{m}(t),\mathcal{R}\right\} \]

where:

  • \(\theta_{h}(t)\): head orientation representing attention direction.
  • \(\mathbf{u}_{b}(t)\): upper-body posture describing behavioral engagement.
  • \(\mathbf{x}_{m}(t)\): hand movement representing manipulation behavior.
  • \(\mathcal{R}\): task-related areas of interest.

This representation extends conventional gaze-based assessment by incorporating both attention orientation and physical behavior.

2. Task-related Attention Demand

The allocation of attention across different task regions is quantified as:

\[ A_k(t)=\frac{1}{t}\int_0^t p_k(u)v(u)\,du \]

where \(p_k(u)\) indicates the attention state associated with AOI \(k\), and \(v(u)\) denotes valid attention observations.

The formulation describes:

  • AED: Assembly Execution Demand.
  • ICD: Instruction Checking Demand.
  • TID: Tool/Part Identification Demand.
  • CD: Robot Collaboration Demand.

Higher values indicate greater attention allocation toward the corresponding task requirement.

3. Context-aware Attention Switching Demand

Attention switching introduces additional cognitive demand in HRC because operators must redirect attention between task execution, information processing, and robot interaction.

The switching cost is defined as:

\[ c(a,b)= \begin{cases} 0,&a=b\\ c_{\mathrm{high}},&\{a,b\}=\{\mathrm{ICD},\mathrm{AED}\}\\ c_{\mathrm{int}},&\{a,b\}=\{\mathrm{CD},\mathrm{TID}\}\\ c_{\mathrm{base}},&\text{otherwise} \end{cases} \]

The context-aware switching demand is:

\[ ASD_{ctx}(t)=1-\exp\left(-\frac{C_{sw}(t)}{t}\right) \]

Higher values indicate more frequent or more cognitively demanding attention reallocation.

4. Operation Hesitation Demand

During robot collaboration, operators may temporarily reduce hand movement due to waiting, uncertainty, or coordination requirements.

\[ OHD(t)=\frac{T_{hes}(t)}{T_{collab}(t)} \]

where \(T_{hes}(t)\) represents hesitation duration and \(T_{collab}(t)\) represents valid robot collaboration duration. Higher OHD indicates increased interaction-related coordination difficulty.

Experimental Setup

The workstation layout was organized into four functional regions, including storage, assembly, instruction and robot-collaboration areas, enabling the characterization of different task demands and human–robot interaction states during assembly.
